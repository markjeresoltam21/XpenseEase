## üì± Screenshot Guidelines

### Device & Settings
- **Device**: Use a standard smartphone (iPhone or Android)
- **Orientation**: Portrait mode for all screenshots
- **Status Bar**: Include status bar for authenticity
- **Quality**: High resolution (at least 1080px width)

### Content Guidelines
- **Demo Data**: Use realistic but anonymized data
- **Amounts**: Use varied expense amounts (e.g., $15.50, $42.30, $8.75)
- **Categories**: Show different expense categories
- **Dates**: Use recent dates for relevance

### File Naming Convention
- Use lowercase with hyphens
- Be descriptive but concise
- Match the names specified in README.md exactly

### Image Specifications
- **Format**: PNG (for crisp UI elements)
- **Size**: 400-800px width (for web display)
- **Background**: Include app background/theme

## üìù Taking Screenshots

### Method 1: Device Screenshots
1. Run the app on your physical device via Expo Go
2. Navigate to each screen
3. Take screenshots using device's built-in screenshot function
4. Transfer files to computer
5. Rename according to guidelines above

### Method 2: Emulator Screenshots
1. Run app on Android/iOS simulator
2. Use simulator's screenshot functionality
3. Save directly to screenshots folder

### Method 3: Expo Dev Tools
1. Use Expo's built-in screenshot feature
2. Access via Expo Dev Tools in browser
3. Navigate through app and capture screens

## üîÑ Processing Screenshots

### Editing Steps
1. **Crop**: Remove unnecessary parts, focus on app content
2. **Resize**: Optimize for web display (400-800px width)
3. **Compress**: Reduce file size while maintaining quality
4. **Annotate**: Add highlights or callouts if needed (optional)

### Tools Recommended
- **Free**: GIMP, Paint.NET, or built-in tools
- **Paid**: Adobe Photoshop, Sketch
- **Online**: Canva, Figma (free tier)
